# Profile File Schemas

Exact YAML schemas for each file generated by /profile. All string values MUST be double-quoted. The profile-generate sub-skill uses these schemas as contracts.

---

## profile.yaml (required)

```yaml
name: "{machine-safe-name}"          # lowercase, hyphens only, no spaces
description: "{human-readable}"       # 1-sentence domain summary
version: "1.0"

identity:
  purpose: "{agent purpose statement}"
  domain: "{domain label}"
  focus_areas:                        # 2-4 items
    - "{area}"

config_overrides:
  data_layers:                        # 1+ items, user-provided
    - "{LayerName}"
  research:
    primary: "{pubmed|arxiv|semantic_scholar|openalex|web-search}"
    fallback: "{backend}"
    last_resort: "web-search"
  literature:
    sources: [{backend}, ...]         # subset of [pubmed, arxiv, semantic_scholar, openalex]
    default: "all"
    enrichment:
      enabled: [crossref, unpaywall]
      timeout_per_doi: 5

env_vars:
  required:                           # keys = env var names, values = human description
    VAR_NAME: "description"
  optional:
    VAR_NAME: "description"
```

**Constraints:**
- `name` must match the directory name
- `name` must be unique across `discover_profiles()` results
- `config_overrides.data_layers` must have at least 1 entry
- `config_overrides.research.primary` must be one of: pubmed, arxiv, semantic_scholar, openalex, web-search
- `config_overrides.literature.sources` must be a subset of [pubmed, arxiv, semantic_scholar, openalex]

## confounders.yaml (required)

```yaml
# Top-level keys = data layer names (must match profile.yaml data_layers)
LayerName:
  - "confounder description"

biological_confounders:
  - "confounder"

data_reality_signals:
  species_column: "{column_name or null}"
  rules:
    - condition: "{condition-tag}"
      claim: "{prose claim template}"
```

**Constraints:**
- Every key except `biological_confounders` and `data_reality_signals` must correspond to a data_layer in profile.yaml
- `biological_confounders` is a flat list of strings
- `data_reality_signals.rules` entries need `condition` and `claim` keys

---

## heuristics.yaml (required)

```yaml
file_extensions:
  ".ext": "LayerName"

tool_references:
  "ToolName": "LayerName"
```

**Constraints:**
- All values must reference data layer names from profile.yaml
- Extensions must start with "."
- Tool names are case-sensitive

---

## pii_patterns.yaml (required)

```yaml
column_patterns:
  - "regex_pattern"
```

**Constraints:**
- Each entry is a valid Python regex string
- Patterns should use word boundaries (`\b`) where appropriate
- Minimum 1 pattern

---

## palettes.yaml (optional)

```yaml
labs:
  lab_name:
    - "#HEXVAL"

semantic:
  variable_name:
    value_label: "#HEXVAL"
```

**Constraints:**
- Hex values must be 7 characters: # + 6 hex digits
- Lab palette should have 6-10 colors
- If user skips palette, use Wong 2011 colorblind-safe default:
  `["#E69F00", "#56B4E9", "#009E73", "#F0E442", "#0072B2", "#D55E00", "#CC79A7", "#000000"]`
